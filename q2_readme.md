# Q2: Disentangled Representation Learning for Environment-agnostic Speaker Recognition

I implement a reduced version of the proposed disentanglement model and compare it against a baseline speaker embedding system.

## Requirements

- Ubuntu 22.04+ (or any Linux)
- Python 3.8+
- Virtual environment (recommended)

**Install the required packages:**

```bash
pip install torch==2.2.0 torchaudio==2.2.0 numpy scipy matplotlib pandas scikit-learn tqdm
Setup
Clone the repository (or place the code in a folder, e.g., ~/ques2).

**Prepared the data**
Download the LibriSpeech train-clean-5 subset from OpenSLR and extract it.
Optionally, download MUSAN noise (http://www.openslr.org/resources/17/musan.tar.gz) for realistic additive noise.
Update the paths in configs/config.py to point to your data directories.

Example config (configs/config.py):

python
librispeech_root = '/root/ques2/LibriSpeech/train-clean-5'
noise_root = None        # set to MUSAN path if available
Create the results directory (the script will create it automatically):

bash
mkdir -p results
Training
Run the training script to train both the baseline speaker embedding model and the proposed disentanglement model:

bash
python train.py
The script will:

Load the dataset (clean and noisy versions created on‑the‑fly)

Train the baseline LSTM‑based speaker embedder for 20 epochs

Train the disentanglement model (with reconstruction, adversarial, and speaker classification losses) for 20 epochs

Save the checkpoints as results/baseline.pth and results/proposed.pth

Training logs show the loss per epoch for each model.

**Evaluation**
After training, evaluate the models using:

bash
python eval.py
This script:

Loads the same dataset (for simplicity, we use the training set; in a real scenario a separate test set should be used)

Extracts embeddings from each model

Computes the pairwise cosine similarity between utterances

Calculates the Equal Error Rate (EER) for speaker verification

The results are printed to the console and saved in results/eer_table.txt.

**Results**
Below are the obtained EER values from our experiments:

Model	EER
Baseline	0.0433
Proposed	0.1342
Interpretation
The baseline model (a simple LSTM‑based speaker embedder) achieves a much lower EER than the proposed disentanglement model. This is expected because:

Our proposed model is a reduced reproduction that may require more tuning (e.g., loss weights, model size) to achieve competitive performance.

The dataset (LibriSpeech train‑clean‑5) is small, and the disentanglement task (separating speaker and environment) may be too complex given the limited data.

Nevertheless, the proposed method successfully trains a model that can reconstruct mel‑spectrograms and disentangle environment features, as evidenced by the adversarial loss decreasing during training.

Reproducing the Experiments
Set the paths in configs/config.py to match your data location.

Run python train.py – wait for training to finish (approx. 20–30 minutes on CPU).

Run python eval.py – observe the printed EER values.

**Notes**
The dataset uses fixed‑length mel‑spectrograms (200 frames, ≈2 seconds) to allow batching.

The baseline model is a simple LSTM; you may replace it with a more powerful model (e.g., ECAPA‑TDNN) if desired.

The proposed model includes a decoder that reconstructs the input mel‑spectrogram, which helps learn disentangled representations.
